# 🗺️ 二、 二期敏捷迭代开发路线图 (Sprints Roadmap)

项目在第一阶段已高质量完成了全栈 P0 闭环功能。本二期路线图重点聚焦对 **P1** 和 **P2** 级未实现需求的规划编排，共分 4 个双周 Sprint（共 8 周）。

### 🟢 Sprint 1: 双链脑图与精细化记忆 (Week 1-2)
* **Sprint 目标**：拓展第二大脑，攻克知识库的双链联想与语音辅助复习。
* **开发任务**：
  * **Zettelkasten 笔记 [P1]**：扩展 TipTap 编辑器，支持双向链接 `[[` 自动关联和语法高亮；引入 `React Flow` 动态渲染笔记关联关系图谱。
  * **AI 闪卡生成器 [P1]**：在笔记区加入划词监听，调用大模型 API 自动根据词句生成对应的 Q&A 闪卡并写入本地数据库。
  * **睡前主动回忆 [P2]**：开发夜间回忆组件，卡片背面增加高斯模糊遮罩；集成 `Web Speech API` 进行语音识别记录用户回答，并与标准答案做文本对比。
* **🧠 学习里程碑**：
  * **表达**：录制 3 分钟英文短片，讲解 Zettelkasten 笔记与闪卡生成的联动思路。
  * **编码**：掌握 ProseMirror 核心概念与扩展开发，处理 Web Speech API 浏览器兼容性。

### 🟡 Sprint 2: 多维执行力与智能排期 (Week 3-4)
* **Sprint 目标**：打破习惯独立性状态，实现行为链依赖与 AI 主动任务编排。
* **开发任务**：
  * **习惯捆绑链 [P1]**：引入 `dnd-kit` 支持拖拽排序及编排；在 `db.ts` 中升级 `habits` 表并支持前置习惯约束校验（“喝完咖啡” -> “英语听力”解锁高亮）。
  * **AI 每日任务生成 [P2]**：使用 RAG/Agent 架构 RAG 检索 IndexedDB 历史打卡率和能量值，并在 Dashboard 上渲染流式骨架屏及 AI 排程任务列表。
  * **社交/健身特化记录 [P2]**：新增特化数据记录表单（组数、重量等数据），集成 `uploadthing` 实现打卡凭证图片上传。
* **🧠 学习里程碑**：
  * **社交**：在社群中发布技术手记《如何基于离线优先设计多习惯依赖调度系统》。
  * **前端**：深入掌握跨端 Drag and Drop 碰撞算法与 Supabase 多表级联。

### 🔴 Sprint 3: AI 社交模拟与 RAG 学习辅助 (Week 5-6)
* **Sprint 目标**：攻克关键技术壁垒，让 AI 从单向分析升级为深度角色扮演演练。
* **开发任务**：
  * **社交话术演练 [P1]**：构建 Chat 气泡 UI 架构，后端由大模型 Roleplay 人设驱动，支持 3 回合实战，并在对话后输出情商/边界感多维打分。
  * **Vibe Coding 助手 [P2]**：基于 Supabase pgvector 向量化技术文档与笔记，当代码报错时自动检索匹配并进行 Bug 修复和 Diff 显示。
  * **每日灵魂复盘升级 [P1]**：利用目前灵魂复盘机制，让 AI 结合全天行为记录主动发起极具针对性的问答博弈。
* **🧠 学习里程碑**：
  * **AI Agent**：掌握 pgvector 数据库的部署、RAG Prompt Engineering 和多状态智能体设计。

### 🔵 Sprint 4: 游戏化反馈与云端备份上线 (Week 7-8)
* **Sprint 目标**：实现游戏化激励反馈，打通云端同步多端闭环，正式上线。
* **开发任务**：
  * **Streak 里程碑与徽章 [P2]**：设计徽章展示墙，打卡达 7/30/100 天时配合 `canvas-confetti` 全屏抛撒花雨解锁特殊徽章。
  * **PWA 支持与性能调优**：编写 Service Worker 策略实现全面离线可用，优化 Largest Contentful Paint (LCP) 指标。
  * **云端多端同步与部署**：构建 IndexedDB 与 Supabase 后端增量冲突合并策略（CRDTs）。
* **🧠 学习里程碑**：
  * **包装**：提炼一份 GrowthOS 全栈系统架构图，编写中英双语 README 并在 GitHub 正式开源求职。

# 🛠️ 三、 技术选型与架构决策 (防坑指南)

| 领域 | 推荐选型 | 为什么这么选？ |
| :--- | :--- | :--- |
| **框架** | Next.js 15 (App Router) | 前端主流，支持 Server Components，对 SEO 和首屏加载友好。 |
| **UI 库** | Tailwind CSS + shadcn/ui | 2026年最标准的前端审美，代码可控，极易实现高级暗黑模式。 |
| **状态管理** | Zustand + Immer | 比 Redux 轻量，适合处理复杂的表单和打卡状态，Immer 解决不可变数据痛点。 |
| **本地存储** | Dexie.js (IndexedDB) | 实现 Local-first 的关键，比 localStorage 强大，支持索引和大量数据。 |
| **后端/DB** | Supabase | 提供 Auth, Postgres, Realtime, Edge Functions, Vector (pgvector)，一人全栈首选。 |
| **AI Agent** | Mastra (TS 原生) 或 LangGraph | **Mastra** 对前端/TS 开发者极度友好；**LangGraph** 适合构建复杂的有状态多智能体工作流。 |
| **富文本** | TipTap | 基于 ProseMirror，扩展性极强，做 `[[双向链接]]` 和 `/` 斜杠命令的唯一解。 |
| **动效** | Framer Motion | React 生态最强动效库，让你的应用摆脱“塑料感”。 |
