# 🗺️ 二、 8周敏捷开发路线图 (Sprint Roadmap)

这套路线图将**产品开发**与你的**个人学习计划（AI Agent/Vibe Coding/英语）** 完美融合。

### 🟢 Phase 1: 骨架与基建 (Week 1-2) —— "让系统跑起来"

- **Sprint 目标**：搭建全栈基础设施，完成 P0 级核心打卡与 WOOP 闭环。
- **前端任务 (Vibe Coding 实践)**：
  - 使用 `v0.dev` 或 `Cursor` 快速生成 Dashboard 和 Habit Grid 的 UI 代码（实践 Vibe Coding）。
  - 配置 Next.js 15 (App Router) + Tailwind + shadcn/ui。
  - 使用 Zustand 搭建全局状态管理，实现离线打卡（Local-first）。
- **后端任务**：
  - 初始化 Supabase 项目，设计并创建 `habits`, `woop`, `users` 表。
  - 实现基于 Supabase Auth 的魔法链接（Magic Link）登录。
- **🧠 学习里程碑**：
  - **英语**：强迫自己全英文阅读 Supabase 和 Next.js 官方文档。
  - **AI**：了解 Local-first 架构和 CRDT（为后续数据同步打基础）。

### 🟡 Phase 2: 记忆与心流 (Week 3-4) —— "打造深度工作区"

- **Sprint 目标**：攻克 SM-2 算法，实现沉浸式专注与笔记系统。
- **前端任务**：
  - 手写 SM-2 算法的 TypeScript 实现（处理 Ease Factor, Interval, Reps）。
  - 集成 `TipTap` 富文本编辑器，实现基础的 Markdown 和 `[[双向链接]]` 解析。
  - 开发 Flow Space 番茄钟，加入白噪音音频控制。
- **后端任务**：
  - 设计 `cards` 和 `reviews` 表，编写复杂的 SQL 查询（获取今日到期卡片）。
  - 实现笔记的全文检索（Supabase `pg_trgm` 或 `tsvector`）。
- **🧠 学习里程碑**：
  - **表达**：录制一段 5 分钟视频，讲解“SM-2 算法的数学原理与 TS 实现”（费曼技巧）。
  - **编码**：掌握 TipTap 的自定义 Node/Mark 扩展开发。

### 🔴 Phase 3: AI Agent 注入 (Week 5-6) —— "注入灵魂（核心护城河）"

- **Sprint 目标**：使用标准的 OpenAI 协议 API 驱动，让系统从“死工具”变成“活教练”。
- **前端任务**：
  - 开发 Chat Interface，支持 Streaming（流式输出）和 Markdown 渲染。
  - 实现 **Action Cards**（AI 返回的不仅仅是文字，而是可点击的 React 组件，如 `[开始复习]`）。
- **AI / 后端任务 (硬仗)**：
  - **配置大模型 API**：配置标准的 OpenAI / DeepSeek 协议 Endpoint 与密钥。
  - **实现 Tool Calling 模拟**：基于标准 OpenAI 结构化输出或 Function Calling 赋予 AI 虚拟调用数据库的能力。
  - **实现 RAG (检索增强生成)**：将用户的 Zettelkasten 笔记向量化（使用 Supabase pgvector），让 AI 教练基于你的笔记进行提问。
  - **开发“费曼判官” Prompt**：通过 Few-shot prompting 让 AI 准确评估文本的“小白易懂度”。
- **🧠 学习里程碑**：
  - **AI 注入**：彻底搞懂基于 OpenAI 协议的 API 交互、RAG 和 Vector DB 的工程化落地。
  - **社交**：在技术社群（如即刻/V2EX）分享你的 Agent 架构设计，引发讨论。

### 🔵 Phase 4: 数据可视化与打磨 (Week 7) —— "提供情绪价值"

- **Sprint 目标**：用数据可视化提供强烈的“胜任感”，打磨动效。
- **前端任务**：
  - 使用 D3.js 或 Echarts 绘制五维雷达图和能量波动曲线。
  - 使用 Framer Motion 为打卡、页面切换、卡片翻转添加丝滑的物理动效。
  - 配置 PWA (Progressive Web App)，支持添加到手机桌面，配置离线 Service Worker。
- **后端任务**：
  - 编写 Supabase Edge Functions，处理每日凌晨的定时任务（如重置 Streak，生成昨日复盘报告）。
- **🧠 学习里程碑**：
  - **健身**：这周代码量可能较大，严格执行三分化健身计划，保持精力。
  - **前端**：掌握复杂 SVG 动画和 PWA 离线缓存策略。

### 🟣 Phase 5: 部署、开源与求职包装 (Week 8) —— "收获果实"

- **Sprint 目标**：上线、开源、转化为求职/社交资产。
- **任务清单**：
  - 部署到 Vercel，配置自定义域名。
  - 编写 `README.md`（中英双语），绘制系统架构图（Excalidraw）。
  - 录制 1 分钟的 Product Hunt 风格演示视频（实践视频表达）。
  - 撰写技术博客：《我如何用 OpenAI 接口与 Next.js 打造 AI 驱动的个人成长系统》。
- **🧠 学习里程碑**：
  - **社交/求职**：将项目链接发给目标公司的 HR 或技术负责人；在 Twitter/掘金 发布文章，建立个人 IP。

# 🛠️ 三、 技术选型与架构决策 (防坑指南)

| 领域         | 推荐选型                      | 为什么这么选？                                                                          |
| :----------- | :---------------------------- | :-------------------------------------------------------------------------------------- |
| **框架**     | Next.js 15 (App Router)       | 前端主流，支持 Server Components，对 SEO 和首屏加载友好。                               |
| **UI 库**    | Tailwind CSS + shadcn/ui      | 2026年最标准的前端审美，代码可控，极易实现高级暗黑模式。                                |
| **状态管理** | Zustand + Immer               | 比 Redux 轻量，适合处理复杂的表单和打卡状态，Immer 解决不可变数据痛点。                 |
| **本地存储** | Dexie.js (IndexedDB)          | 实现 Local-first 的关键，比 localStorage 强大，支持索引和大量数据。                     |
| **后端/DB**  | Supabase                      | 提供 Auth, Postgres, Realtime, Edge Functions, Vector (pgvector)，一人全栈首选。        |
| **AI 注入**  | 标准 OpenAI 协议 API          | **轻量无依赖**，直接利用 Next.js Route Handlers 对接各类大模型，速度极快且极易部署。     |
| **富文本**   | TipTap                        | 基于 ProseMirror，扩展性极强，做 `[[双向链接]]` 和 `/` 斜杠命令的唯一解。               |
| **动效**     | Framer Motion                 | React 生态最强动效库，让你的应用摆脱“塑料感”。                                          |
