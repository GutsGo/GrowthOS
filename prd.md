# 🚀 GrowthOS 产品需求文档 (PRD) v1.0

## 一、 产品概述

- **产品定位**：一款基于学习科学与 AI Agent 驱动的「自我驱动型成长操作系统」。
- **目标用户**：追求持续进步的知识工作者、独立开发者、终身学习者（首个超级用户是你自己）。
- **核心价值**：将抽象的方法论（WOOP、费曼、间隔重复、原子习惯）固化为**默认交互**，通过 AI Agent 提供主动干预，消除“坚持”带来的意志力损耗。
- **设计哲学**：Local-first（离线优先）、Keyboard-driven（键盘驱动，类似 Raycast）、Dark-mode Only（沉浸式暗黑）。

## 二、 核心功能矩阵 (Feature Matrix)

| 模块                 | 核心功能                                | 对应方法论          | 前端核心组件 (shadcn/ui)        |
| :------------------- | :-------------------------------------- | :------------------ | :------------------------------ |
| **🎯 规划 (Plan)**   | WOOP 每日意图设定、能量曲线预测         | WOOP / 能量管理     | `Dialog`, `Slider`, `Form`      |
| **⚡ 执行 (Act)**    | 习惯打卡、番茄钟心流、社交/健身记录     | 原子习惯 / 习惯捆绑 | `Toggle`, `Timer`, `Progress`   |
| **🧠 记忆 (Mem)**    | SM-2 间隔重复卡片、双向链接笔记         | 间隔重复 / 卡片盒   | `Card`, `Flip-Card`, `Graph`    |
| **🗣️ 输出 (Out)**    | 费曼语音转文字、AI 表达清晰度评分       | 费曼技巧 / 双重编码 | `AudioRecorder`, `Markdown`     |

| **📊 复盘 (Review)** | 多维能力雷达图、GitHub 式热力图         | 进度可视化          | `RadarChart`, `CalendarHeatmap` |

## 三、 信息架构 (Information Architecture)

采用经典的 **左侧导航 + 顶部全局搜索 (Cmd+K) + 右侧主内容区** 布局。

```text
GrowthOS
├── 🏠 Dashboard (今日指挥舱)
├── ⏱️ Flow Space (心流与费曼输出)
├── 🧠 Brain (第二大脑 / 卡片与笔记)

└── ⚙️ Settings (API 配置 / 习惯管理)
```

## 四、 核心页面 UI 草图与交互描述

### 4.1 全局视觉与布局规范

- **色彩系统**：
  - 背景：`#09090b` (Zinc-950)
  - 卡片/面板：`#18181b` (Zinc-900) + 1px `#27272a` 边框
  - 主强调色（生长/习惯）：`#10b981` (Emerald-500)
  - 次强调色（AI/逻辑）：`#3b82f6` (Blue-500)
- **布局尺寸**：左侧 Sidebar 宽 `240px`（可折叠至 `64px`），顶部 Header 高 `64px`。
- **全局快捷键**：`Cmd/Ctrl + K` 呼出全局命令面板（快速打卡、新建卡片、呼叫 AI）。

---

### 4.2 首页 Dashboard (今日指挥舱)

**设计目标**：一眼看清今日任务，3 次点击内完成核心打卡。

**UI 布局描述**：

- **顶部 (Header Area)**：
  - 左侧：日期与问候语（如："Sunday, May 24. 你的连续打卡记录：42 天 🔥"）。
  - 右侧：当前能量状态滑块（1-10级，AI 会根据此推荐任务）。
- **主体 (Grid Layout: 2列布局)**：
  - **左列 (占 60%)**：
    - **WOOP 意图卡**：展示今日 Wish 和 if-then Plan。若未填写，显示高亮引导按钮。
    - **习惯矩阵 (Habit Grid)**：网格排列的打卡按钮（英语、AI编码、三分化健身、社交）。点击后触发微动效（如撒花或绿色光晕），并更新底部 Streak 进度条。
  - **右列 (占 40%)**：
    - **间隔重复队列 (Review Queue)**：显示 "今日到期 15 张卡片"。点击 "Start Review" 进入全屏专注模式。
    - **AI 教练洞察 (Coach Insight)**：一个带有 AI 头像的卡片，显示一句基于昨日数据的锐评（如："你昨天社交表达训练缺席了，今晚 21:00 需要我陪你做个模拟演练吗？"），附带 `[Accept]` 和 `[Snooze]` 按钮。

---

### 4.3 专注空间 Flow Space (心流与费曼)

**设计目标**：屏蔽干扰，深度输出。

**UI 布局描述**：

- **全屏沉浸式界面**，隐藏侧边栏。
- **中心区域**：
  - 一个极简的番茄钟（大字体倒计时，如 `24:59`）。
  - 下方是 **费曼输出板 (Feynman Board)**：
    - 输入框："今天我要把 [ AI Agent 的 ReAct 模式 ] 讲给小白听。"
    - 文本区/语音录入区：支持 Markdown 实时渲染。
    - **[AI 判官] 按钮**：点击后，AI 分析文本，返回 "小白易懂度评分 (0-100)" 和 "卡壳点提示"（如："你在解释 Tool Calling 时使用了太多术语，建议用'点菜'来类比"）。
- **右侧悬浮抽屉**：白噪音控制器（雨声/咖啡馆）+ 当前任务关联的 Zettelkasten 笔记卡片。

---

### 4.4 知识图谱 Brain (第二大脑)

**设计目标**：建立知识连接，而非单纯收藏。

**UI 布局描述**：

- **三栏布局 (类似 Obsidian/Notion)**：
  - **左栏 (目录/标签)**：按领域分类（Frontend, AI Agent, Social Skills, English）。
  - **中栏 (编辑器)**：块级编辑器 (Block Editor)，支持 `[[双向链接]]` 和 `/` 斜杠命令。
  - **右栏 (AI 辅助与图谱)**：
    - **局部知识图谱 (Graph View)**：使用 D3.js 或 Cytoscape.js 渲染当前笔记的关联节点。
    - **AI 卡片生成器**：点击 "Extract to Anki"，AI 自动将当前笔记的核心概念提取为正反面卡片，预览后一键存入间隔重复库。

---

### 4.5 AI 教练舱 Coach AI (Agent 交互)

**设计目标**：不是普通的 Chatbot，而是带有**工具调用 (Tool Use)** 和**行动卡片 (Action Cards)** 的智能体。

**UI 布局描述**：

- **对话流界面**，但消息气泡中嵌套可交互组件。
- **特色交互场景**：
  - **场景 A (社交演练)**：
    - 用户："我想练习怎么和女生开启话题，设定对方是刚认识的插画师。"
    - AI (Roleplay)：发送一条模拟消息。
    - 用户回复后，AI 切换为 "Coach 视角"，给出反馈卡片：`[情绪价值: 8/10] [需求感: 3/10] [改进建议: ...]`。
  - **场景 B (主动干预)**：
    - AI 发送一张 **Action Card**："检测到你今天 AI 编码时间不足 1 小时。我为你生成了一个 15 分钟的 LangGraph 小挑战。"
    - 卡片底部有 `[Start Challenge]` 按钮，点击直接跳转到 Flow Space。

---

### 4.6 数据复盘 Stats (多维度雷达)

**设计目标**：用数据可视化提供强烈的「胜任感」和「关联感」。

**UI 布局描述**：

- **顶部**：GitHub 风格的年度热力图 (Calendar Heatmap)，颜色深浅代表当日「习惯完成率 + 专注时长」的综合指数。
- **中部 (核心)**：**五维能力雷达图 (Radar Chart)**。
  - 维度：`技术深度 (AI/代码)`、`语言广度 (英语)`、`身体机能 (健身)`、`表达魅力 (输出)`、`社交连接 (聊天)`。
  - 交互：支持拖动时间轴，对比 "本月 vs 上月" 的雷达图面积变化。
- **底部**：WOOP 成功率折线图、每周能量波动曲线。

## 五、 AI Agent 核心工作流设计 (Backend/Agent Layer)

这是系统的灵魂，建议使用 **LangGraph** 或 **Mastra** 构建状态机。

### Agent 1: The Reviewer (每日复盘 Agent)

- **触发条件**：每晚 21:30 定时触发，或用户手动点击 "Daily Review"。
- **输入上下文**：今日习惯打卡状态、WOOP 完成度、费曼输出文本、专注时长。
- **工作流 (Graph)**：
  1. `Analyze Node`：分析数据，找出亮点和缺失（如：健身做了，但英语没看）。
  2. `Generate Questions Node`：生成 3 个直击灵魂的复盘问题。
  3. `Action Suggestion Node`：如果某项习惯连续 miss 2 天，自动生成 "降低门槛" 的 if-then 计划建议。
- **输出**：推送到 Dashboard 的 Coach Insight 卡片。

### Agent 2: The Social Sparring Partner (社交陪练 Agent)

- **System Prompt**：你是一个高情商社交教练。你需要扮演特定人设与用户聊天，并在每 3 个回合后跳出角色，对用户的「情绪捕捉」、「幽默感」、「边界感」进行打分和指导。
- **工具 (Tools)**：`save_insight_to_brain` (将好的话术自动保存到笔记库)。

## 六、 开发里程碑 (Roadmap)

作为独立开发者，建议采用 **敏捷迭代**，避免陷入"过度设计"的陷阱。

| 阶段                    | 周期     | 核心交付物 (Deliverables)                                                  | 你的技术收益                                              |
| :---------------------- | :------- | :------------------------------------------------------------------------- | :-------------------------------------------------------- |
| **Phase 1: MVP**        | Week 1-2 | Dashboard、习惯打卡、WOOP、基础 SM-2 卡片。数据存 LocalStorage/IndexedDB。 | 跑通 Next.js 15 + shadcn/ui + Zustand 状态管理。          |
| **Phase 2: 记忆与输出** | Week 3-4 | Flow Space (费曼板)、Brain (基础 Markdown 笔记)、热力图与雷达图。          | 掌握 TipTap 富文本编辑器、D3.js/ECharts 复杂可视化。      |
| **Phase 3: AI 注入**    | Week 5-6 | 接入 Mastra/LangGraph，实现 Coach Agent 和社交陪练 Agent，打通 API。       | **真正掌握 AI Agent 工程化、Tool Calling、RAG。**         |
| **Phase 4: 云端与发布** | Week 7-8 | 接入 Supabase (Auth + DB)，实现多端同步，部署到 Vercel，撰写发布文章。     | 掌握全栈架构、OAuth、云端数据同步，**产出求职王牌项目**。 |

## 七、 给前端开发的特别 Tips (避坑指南)

1. **别一开始就搞复杂的富文本**：MVP 阶段直接用 `textarea` + `marked.js` 预览，Phase 2 再上 TipTap。
2. **状态管理**：这种表单和打卡极多的应用，推荐使用 **Zustand** 结合 **immer**，比 Redux 轻量且好写。
3. **动画是灵魂**：打卡成功时的反馈、卡片翻转的 3D 效果、雷达图展开的动画，请使用 **Framer Motion**，这会让你的应用看起来像价值 $20/月的 SaaS，而不是学生作业。
4. **离线优先 (Local-first)**：使用 **Dexie.js** (IndexedDB 封装) 作为本地数据库，后续通过 Supabase 做后台静默同步。这能极大提升应用的"跟手感"。
